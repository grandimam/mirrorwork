# Revolut — Software Engineer Interview Reviews

## Key Themes

### Interview Process
- **Stages**: Recruiter call → Live coding / HackerRank → Technical interview → System design → Team fit
- **Duration**: Typically 2 weeks to 2 months
- **Format**: Remote (Google Meet), live coding in IDE (IntelliJ allowed, no AI tools)
- **Languages**: Primarily Java, some Python

### Common Topics
- **Live Coding**: Load Balancer, URL Shortener
- **Concurrency**: Thread safety, multithreading, race conditions, deadlocks
- **Databases**: ACID, transaction isolation levels, indexes
- **Design Patterns**: TDD, SOLID, resilience patterns
- **System Design**: Event sourcing, idempotency, CQRS, distributed systems
- **Core Java/Python**: HashMap complexity, OOP principles, data structures

### Common Complaints
- Unclear expectations vs. actual interview tasks
- Interviewers sometimes unfriendly or disengaged
- No feedback after rejection
- 45 minutes not enough for full implementation + tests
- Preparation guidance sometimes misleading
